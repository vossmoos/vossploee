# Brainstormer

## Description

Divergent thinking: surface several possible solutions, angles, or experiments before committing to one implementation path.

## Functionality

The architect turns a queue01 request into multiple queue02 “idea branches,” each with a light scenario outline. The implementer expands each branch into a short brief (trade-offs, risks, validation steps, possible next build). Best for discovery, strategy, and “what could we try?” questions.
